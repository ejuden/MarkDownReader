Troubleshooting

Document appears blank

If a document opens but nothing is displayed:

  • Verify the file is a valid text file with Markdown content.
  • Check that the file uses UTF-8 encoding. Other encodings may not render correctly.
  • Try closing and reopening the file.

Double-clicking .md files opens a different app

You need to set MarkDown Reader as the default handler:

  1. Right-click any .md file in Finder.
  2. Choose Get Info.
  3. Under "Open with:", select MarkDown Reader.
  4. Click Change All... and confirm.

Images not loading

MarkDown Reader can display:

  • Remote images (URLs starting with http:// or https://) — requires an internet connection.
  • Relative images — images referenced with relative paths within the same directory as the Markdown file.

If remote images aren't loading, check your internet connection. If local images aren't showing, make sure the image path in the Markdown is correct relative to the document.

Export PDF is blank or incomplete

If your exported PDF appears blank:

  • Wait a moment for the document to fully render before exporting.
  • If the document contains remote images, ensure they have finished loading.
  • Try printing to PDF instead (using ⌘P and selecting "Save as PDF").

Table of contents is empty

The table of contents is generated from headings in the document. If it's empty, the document may not contain any headings (lines starting with #).

Reporting Issues

If you encounter a bug or have a feature request, please visit the GitHub repository and open an issue. Include:

  • Your macOS version
  • A description of the problem
  • The Markdown content that triggered the issue (if applicable)

Supported Markdown

MarkDown Reader supports standard Markdown syntax along with common GitHub Flavored Markdown (GFM) extensions.

Text Formatting

SyntaxResult
**bold**bold
*italic*italic
***bold italic***bold italic
~~strikethrough~~strikethrough
`inline code`inline code

Headings

Use # through ###### for heading levels 1 through 6. Setext-style headings (underlined with === or ---) are also supported.

Links & Images

SyntaxDescription
[text](url)Hyperlink
[text](url "title")Link with hover title
![alt](url)Image
<https://...>Autolink

Lists

Unordered lists use -, *, or + as markers. Ordered lists use numbers followed by . or ). Indent with two spaces for nested lists.

Task lists use - [ ] for unchecked and - [x] for checked items.

Code Blocks

Fenced code blocks use triple backticks (```) or triple tildes (~~~). You can specify a language after the opening fence for labeling:

```python
def hello():
    print("Hello, world!")
```

Tables

GFM-style tables with optional alignment:

| Left | Center | Right |
|:-----|:------:|------:|
| A    |   B    |     C |

Other Elements

  • Blockquotes — lines starting with >, with nesting supported.
  • Horizontal rules — three or more ---, ***, or ___.
  • Line breaks — end a line with two spaces for a hard line break.
  • Escape characters — use \ before special characters to display them literally.
